# Bytecode VM Documentation

## Overview

This is a custom bytecode virtual machine designed to verify CTF flags by:
1. Reading the input flag at runtime
2. Constructing L and D matrices from the flag
3. Computing A = L Ă— D Ă— L^T using matrix operations
4. Comparing with a hardcoded expected matrix
5. Returning success (0) or failure (1)

## Architecture

### VM Structure (`bytecode_vm.h`)

```c
struct BytecodeVM {
    double stack[256];           // Stack for operands
    int sp;                      // Stack pointer
    double regs[16];             // General purpose registers
    double matrices[8][9][9];    // 8 matrices, each 9x9
    double memory[4096];         // Additional memory
    char flag[256];              // Input flag
    // ... execution state
}
```

### Instruction Set

**Stack Operations:**
- `OP_PUSH_IMM` (0x01) - Push double immediate
- `OP_POP` (0x02) - Pop from stack
- `OP_DUP` (0x03) - Duplicate top

**Register Operations:**
- `OP_LOAD_REG` (0x04) - Load register to stack
- `OP_STORE_REG` (0x05) - Store stack to register

**Arithmetic:**
- `OP_ADD`, `OP_MUL`, `OP_SUB`, `OP_DIV`, `OP_NEG`

**Matrix Operations:**
- `OP_MAT_INIT_IDENTITY` (0x10) - Initialize identity matrix
- `OP_MAT_SET` (0x11) - Set element [i][j]
- `OP_MAT_GET` (0x12) - Get element [i][j]
- `OP_MAT_MUL` (0x13) - Multiply two matrices
- `OP_MAT_TRANS` (0x14) - Transpose matrix
- `OP_MAT_CMP` (0x15) - Compare matrices for equality

**Control Flow:**
- `OP_JMP` (0x20) - Unconditional jump
- `OP_JEQ`, `OP_JNE` - Conditional jumps

**Flag I/O:**
- `OP_FLAG_LEN` (0x32) - Push flag length
- `OP_PUSH_FLAG_CHAR` (0x31) - Push flag[index]

**Special:**
- `OP_EXIT` (0xFF) - Exit with code from stack

## Files

### Core VM Files
- **bytecode_vm.h** - VM struct and instruction definitions
- **bytecode_vm.c** - VM implementation (8.8 KB)
  - Stack and register management
  - Matrix operations (multiply, transpose, compare)
  - Instruction execution loop
  - Floating-point comparison with tolerance

- **bytecode_runner.c** - CLI for running bytecode
  - Reads bytecode file
  - Accepts flag as command-line argument
  - Outputs success/failure

### Bytecode Generation
- **bytecode_gen.py** - Generates bytecode from verification logic
  - Creates bytecode that reads flag at runtime
  - Unrolls matrix construction loops
  - Encodes expected A matrix
  - Generates matrix computation sequence

### Example Bytecode
- **bytecode.bin** - Generated bytecode (2.3 KB)
  - Pre-generated for the known flag
  - Can be used with `bytecode_runner`

## Usage

### Generate Bytecode
```bash
python3 bytecode_gen.py [output_file]
# Outputs: bytecode.bin (default)
```

The bytecode is independent of any specific flag - it reads and verifies dynamically.

### Run with Bytecode
```bash
./bytecode_runner bytecode.bin <flag>
# Returns: 0 if flag is correct, 1 if incorrect
# Outputs: "SUCCESS: Flag is correct!" or "FAILURE: Flag is incorrect."
```

### Compile
```bash
# Compile VM and runner
gcc -o bytecode_runner bytecode_vm.c bytecode_runner.c -lm
```

## Algorithm

### Bytecode Execution Flow

1. **Initialize matrices** - L and D as identity matrices
2. **Extract flag length** - Store in register
3. **Fill L matrix** - Read bytes from flag, apply offset, place in lower triangle
   - Diagonal skipping: when `r == c`, reset `c = 0` and increment `r`
   - Continues until `r == 9`
4. **Fill D matrix** - Read last 9 characters from flag, apply offset, place on diagonal
5. **Hardcode expected A** - Pre-computed matrix for the known flag
6. **Compute A** - Execute: `A = (L Ă— D) Ă— L^T`
   - Uses matrix IDs 4 and 5 as temporary storage
7. **Compare** - Check if computed A matches expected A with floating-point tolerance (Îµ = 1e-9)
8. **Exit** - Return 0 if match, 1 if not

### Constants

- `OFFSET = -77` - Applied to all character codes
- `N = 9` - Matrix dimension and last N characters for D
- `EPSILON = 1e-9` - Floating-point comparison tolerance

## Performance

- **Bytecode size**: ~2.3 KB (uncompressed)
- **Stack size**: 256 doubles (~2 KB)
- **Register count**: 16
- **Matrix storage**: 8 Ă— 9Ă—9 matrices (~5.2 KB)
- **Execution time**: < 1ms for single flag check

## Security Notes

- This is designed for a CTF reverse engineering challenge
- The bytecode embeds the verification logic but not the flag itself
- The flag is verified at runtime against hardcoded matrix values
- Reverse engineering would require:
  1. Disassembling the bytecode
  2. Understanding the matrix operations
  3. Working backwards from the hardcoded A matrix
